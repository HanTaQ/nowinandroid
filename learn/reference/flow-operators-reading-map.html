Flow 操作符源码阅读速查

读 Now in Android 时，先看 Flow 的“外层类型”怎么变，再看每次 emission 里面的数据怎么变。

Flow.map

改每次发出的值

Flow<A>.map { A -> B } 得到 Flow<B>。它不是遍历列表 item，除非 A 本身就是列表且你在 lambda 里调用 list.map。

List.map

改列表里的每个元素

List<A>.map { A -> B } 得到 List<B>。它发生在内层集合上。

combine

多个 Flow 合成一个值

combine(a, b) { latestA, latestB -> C } 得到 Flow<C>。任意一个上游更新，都会用最新值重新计算。

flatMapLatest

上游变，切换下游 Flow

上游每次发出新值，就创建一个新的内部 Flow，并取消旧的内部 Flow。常用于“条件变化后重新查询”。

stateIn

Flow 变 StateFlow

把一个 Flow 变成有当前值的 StateFlow。UI 更适合收集 StateFlow，因为它需要当前状态和初始状态。

collect

真正开始收集

很多 Flow 是 cold 的。没有 collect，就不会执行上游。stateIn(WhileSubscribed) 也依赖是否有订阅者。

读类型链

Flow<List<UserNewsResource>>
    .map(NewsFeedUiState::Success)
// Flow<NewsFeedUiState.Success>
// 可作为 Flow<NewsFeedUiState> 使用

这里的 map 改的是 Flow 每次发出的“整个列表”，不是列表里的每个 UserNewsResource。
