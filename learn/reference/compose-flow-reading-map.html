Compose 与 Flow 源码阅读速查

读 Now in Android 时，先把这些词当成路标。源码里的复杂片段通常只是这些路标的组合。

Compose State

数据变，UI 重画

Compose 是 declarative UI：UI 函数接收 state，state 变化后触发 recomposition。ViewModel 通常暴露 StateFlow<UiState>，Screen 用 collectAsStateWithLifecycle() 转成 Compose 可观察的 state。

State Hoisting

状态下传，事件上传

组件接收 value 和 onValueChange，自己不保存业务状态。Now in Android 里常见形态是 isSelected 下传，onClick(topicId, checked) 上传。

Modifier

修饰也是布局逻辑

Modifier 可以改变尺寸、布局、外观、输入和语义。链式顺序有意义，越靠前越先包住后面的节点。

Constraints

父组件给范围

Compose 布局不是“想多大就多大”。父组件给子组件 min/max width/height 约束，子组件在约束内测量自己。heightIn(max = ...) 就是在给高度设置上限。

Dp vs Sp

尺寸与字体缩放

dp 主要用于布局尺寸；sp 会跟随用户字体大小缩放。with(LocalDensity.current) { 240.sp.toDp() } 是把“字体缩放后的 240sp 高度”转换成布局可用的 dp。

Flow.combine

多个来源合成状态

combine(a, b) { latestA, latestB -> uiState } 会取每个 Flow 的最新值，合成一个新 Flow。任意一个来源更新，都会重新计算 UI state。

stateIn

冷流变热状态

stateIn(viewModelScope, started, initialValue) 把 cold Flow 转成 StateFlow，让 UI 随时能读到最后一个值，并且需要一个初始状态。
