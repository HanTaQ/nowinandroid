第 6 课：ForYouViewModel 里的 stateIn 与 UI state

这一课从 DataStore 回到 ViewModel：为什么 Now in Android 把 repository/use case 的 Flow 转成 StateFlow，以及 stateIn 的三个参数怎么读。

先看 ViewModel 暴露了哪些状态

onboarding

onboardingUiState

表示 onboarding 是否加载中、是否显示、显示哪些 topics。

feed

feedState

表示新闻 feed 是 loading，还是已经有 UserNewsResource 列表。

sync

isSyncing

表示后台同步是否正在进行，用于控制顶部 loading indicator。

deep link

deepLinkedNewsResource

表示通过 deep link 打开的新闻资源，处理完后会清空。

为什么要 StateFlow

UI 需要的是“随时能读到当前值”的状态。普通 Flow 更像是一条数据管道，collect 时才开始工作；StateFlow 更像一个持续更新的状态容器，有当前值，可以被 Compose 安全收集。

val onboardingUiState: StateFlow<OnboardingUiState> =
    combine(
        shouldShowOnboarding,
        getFollowableTopics(),
    ) { shouldShowOnboarding, topics ->
        if (shouldShowOnboarding) {
            OnboardingUiState.Shown(topics = topics)
        } else {
            OnboardingUiState.NotShown
        }
    }
        .stateIn(
            scope = viewModelScope,
            started = SharingStarted.WhileSubscribed(5_000),
            initialValue = OnboardingUiState.Loading,
        )

stateIn 的三个参数

scope

viewModelScope

这个共享状态的生命周期跟 ViewModel 绑定。ViewModel 清理时，相关协程也会取消。

started

WhileSubscribed(5_000)

有 UI 订阅时开始收集上游；UI 暂时停止订阅后，延迟 5 秒再停止，避免配置变化等短暂停顿导致立刻重启上游。

initialValue

Loading / false / null

上游 Flow 还没发出第一个值时，UI 先看到一个合理的初始状态。

不同状态的 initialValue

  • onboardingUiState 初始为 OnboardingUiState.Loading，因为 topics/userData 还没组合出来。
  • feedState 初始为 NewsFeedUiState.Loading，因为新闻 feed 还没加载出来。
  • isSyncing 初始为 false，因为默认先认为没有同步中。
  • deepLinkedNewsResource 初始为 null，因为默认没有 deep link 新闻。

stateIn 和测试

ForYouViewModelTest 里多次出现：

backgroundScope.launch(UnconfinedTestDispatcher()) {
    viewModel.onboardingUiState.collect()
}

这是因为 stateIn 使用了 SharingStarted.WhileSubscribed。如果没有订阅者，上游 Flow 可能不会开始收集。测试里主动 collect，等于模拟 UI 正在订阅这个 state。

读源码口诀

  1. 看到 map(...).stateIn(...)：一个数据源转换成 UI state。
  2. 看到 combine(...).stateIn(...)：多个数据源合成 UI state。
  3. 看到 initialValue：问 UI 在第一批数据到来前应该显示什么。
  4. 看到 WhileSubscribed：问这个状态什么时候开始/停止收集上游。
