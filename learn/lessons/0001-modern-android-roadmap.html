第 1 课：建立学习地图

这一课的目标很小：你先不急着钻进每个细节，而是知道接下来怎样用 Now in Android 学会现代 Android 应用的五个核心能力。

学习顺序

1. Architecture：先追一条真实用户路径

从 For You 页面开始，追踪 ForYouScreen、ForYouViewModel、GetUserNewsResourcesUseCase、repository、Room/DataStore/network。你会先学会 UDF、Flow、UI state、offline-first。

2. Modularization：再看代码为什么放在这些模块里

对照 settings.gradle.kts 和各模块 README，理解 app、feature:*:api、feature:*:impl、core:* 的边界。重点不是记模块名，而是判断“新代码应该放哪里”。

3. Testing：用测试反推可测试架构

先读 ForYouViewModelTest，再读 repository tests、Compose UI tests、Roborazzi screenshot tests。你会看到官方示例如何用 test doubles 替代 mock-heavy 测试。

4. UI：把 Compose 组件分层读

从 feature screen 到 core:ui 再到 core:designsystem。重点是 state hoisting、Material 3、adaptive layout、theme、navigation 的组织方式。

5. Performance：最后学习如何拿证据优化

读 benchmarks、baseline-prof.txt、Compose compiler metrics 配置。目标是知道启动、重组、稳定性和关键路径分别该用什么工具观察。
