第 13 课：测试两层 combine 生成 Onboarding 状态

上一课故意缺少 topics，所以 Onboarding 停在 Loading。本课补齐两个上游的首个值，观察 use case 与 ViewModel 中的两层 combine 如何生成 Shown。

本课阅读的测试

打开 ForYouViewModelTest.onboardingIsShownWhenNewsResourcesAreLoading()。真实断言逐个构造了三个 FollowableTopic，为了先看清行为，可以把它压缩成下面的等价结构：

@Test
fun onboardingIsShownWhenNewsResourcesAreLoading() = runTest {
    // Arrange：启动两个 stateIn
    backgroundScope.launch(UnconfinedTestDispatcher()) {
        viewModel.onboardingUiState.collect()
    }
    backgroundScope.launch(UnconfinedTestDispatcher()) {
        viewModel.feedState.collect()
    }

    // Act：两个 Fake Repository 都发出首个值
    topicsRepository.sendTopics(sampleTopics)
    userDataRepository.setFollowedTopicIds(emptySet())

    // Assert：下面是对真实长断言的结构化表达
    val expectedTopics = sampleTopics.map { topic ->
        FollowableTopic(topic = topic, isFollowed = false)
    }
    assertEquals(
        OnboardingUiState.Shown(expectedTopics),
        viewModel.onboardingUiState.value,
    )
    assertEquals(
        NewsFeedUiState.Success(emptyList()),
        viewModel.feedState.value,
    )
}

第一层 combine：生成 FollowableTopic

GetFollowableTopicsUseCase 同时需要所有 Topic 和用户关注记录：

combine(
    userDataRepository.userData, // Flow<UserData>
    topicsRepository.getTopics() // Flow<List<Topic>>
) { userData, topics ->
    topics.map { topic ->
        FollowableTopic(
            topic = topic,
            isFollowed = topic.id in userData.followedTopics,
        )
    }
}
// Flow<List<FollowableTopic>>

测试发出的 followedTopics 是空集合，所以每个 topic.id in emptySet() 都是 false。三个 Topic 因而变成三个 isFollowed = false 的 FollowableTopic。

第二层 combine：决定是否显示 Onboarding

ForYouViewModel 再组合两个结果：

combine(
    shouldShowOnboarding, // Flow<Boolean>
    getFollowableTopics() // Flow<List<FollowableTopic>>
) { shouldShow, topics ->
    if (shouldShow) {
        OnboardingUiState.Shown(topics)
    } else {
        OnboardingUiState.NotShown
    }
}
// Flow<OnboardingUiState>

TestUserDataRepository 的当前数据以 shouldHideOnboarding = false 为基础，因此：

shouldShowOnboarding
= !userData.shouldHideOnboarding
= !false
= true

现在第二层 combine 同时得到了 true 和完整的 List<FollowableTopic>，于是产生 OnboardingUiState.Shown(topics)。

把两层数据流连起来

topicsRepository ───────────────┐
                               ├─ combine ─> List<FollowableTopic> ─┐
userDataRepository ────────────┘                                    │
                                                                    ├─ combine
userDataRepository ─> map(!shouldHideOnboarding) ─> Boolean ───────┘
                                                                    │
                                                                    ▼
                                                OnboardingUiState.Shown
源码阅读技巧

看到嵌套 Flow 时，先给每条边标类型，再判断每个 combine 还缺哪个首个值。复杂链路会立刻变得可追踪。

为什么 Feed 仍然是空的 Success

虽然测试没有调用 newsRepository.sendNewsResources()，但用户没有关注任何 Topic。observeAllForFollowedTopics() 会直接产生 emptyList()，无需等待 News Repository，因此 Feed 是“已完成但无内容”的 Success(emptyList())。

测试名称与行为不完全一致

函数名写着 NewsResourcesAreLoading，但当前断言期望的是 Success(emptyList())。从当前代码可以判断，测试名可能没有随实现演进同步更新。阅读测试时，名称用于快速定位意图，真正的行为契约仍由输入和断言决定。
