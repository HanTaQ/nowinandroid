第 10 课：feedState 如何进入 Compose UI

这一课从 ForYouViewModel.feedState 继续往 UI 走，目标是看懂 StateFlow 如何变成 Compose 可渲染的状态。

先反馈第 9 课练习

  • 第 1 题正确：null 表示不启用过滤，emptySet() 表示启用过滤但没有可匹配 id，所以结果为空。
  • 第 2 题正确：DAO 的 SQL 需要 useFilterTopicIds 这个 boolean 来决定是否进入 topic 过滤分支。
  • 第 3 题正确：没有关注 topic 时直接返回 flowOf(emptyList())，可以避免一次没有意义的数据库查询。
  • 第 4 题基本正确：当 query 对应的 Flow 被重新创建并收集时，Room DAO 的 @Query 会重新执行，并且后续数据库表变化也会让这个 Flow 继续发出新结果。

1. ViewModel 把 repository Flow 包成 UI state

ForYouViewModel 里：

val feedState: StateFlow<NewsFeedUiState> =
    userNewsResourceRepository.observeAllForFollowedTopics()
        .map(NewsFeedUiState::Success)
        .stateIn(
            scope = viewModelScope,
            started = SharingStarted.WhileSubscribed(5_000),
            initialValue = NewsFeedUiState.Loading,
        )

上一课已经知道：

observeAllForFollowedTopics()
// Flow<List<UserNewsResource>>

所以这里的 .map(NewsFeedUiState::Success) 等价于：

.map { feed: List<UserNewsResource> ->
    NewsFeedUiState.Success(feed)
}
Kotlin 语法

NewsFeedUiState::Success 是构造函数引用。因为 Success 的构造函数接收一个 List<UserNewsResource>，刚好能作为 map 的转换函数。

2. NewsFeedUiState 用 sealed interface 表达有限 UI 状态

core/ui/NewsFeed.kt 里：

sealed interface NewsFeedUiState {
    data object Loading : NewsFeedUiState

    data class Success(
        val feed: List<UserNewsResource>,
    ) : NewsFeedUiState
}

这里没有直接用 List<UserNewsResource>?，而是用一个 sealed hierarchy，把 UI 状态表达得更清楚：

  • Loading：新闻流还没加载完成。
  • Success(feed)：新闻流加载完成，里面可能有很多项，也可能是空列表。

这样 UI 写 when (feedState) 时，能明确处理每种状态。

3. stateIn 让冷 Flow 变成 ViewModel 持有的 StateFlow

stateIn 在这里做三件事：

  1. 把普通 Flow 转成 StateFlow。
  2. 给 UI 一个立刻可读的初始值：NewsFeedUiState.Loading。
  3. 把收集工作绑定到 viewModelScope，让状态由 ViewModel 持有。

SharingStarted.WhileSubscribed(5_000) 的意思是：有订阅者时开始收集；订阅者消失后等待 5 秒再停止。这个策略适合 UI 短暂离开又回来，比如配置变化或页面切换。

4. 外层 ForYouScreen 收集 ViewModel 状态

feature/foryou/.../ForYouScreen.kt 里有两个同名函数。外层函数负责连接 ViewModel：

@Composable
fun ForYouScreen(
    onTopicClick: (String) -> Unit,
    modifier: Modifier = Modifier,
    viewModel: ForYouViewModel = hiltViewModel(),
) {
    val onboardingUiState by viewModel.onboardingUiState.collectAsStateWithLifecycle()
    val feedState by viewModel.feedState.collectAsStateWithLifecycle()
    val isSyncing by viewModel.isSyncing.collectAsStateWithLifecycle()

    ForYouScreen(
        isSyncing = isSyncing,
        onboardingUiState = onboardingUiState,
        feedState = feedState,
        onTopicCheckedChanged = viewModel::updateTopicSelection,
        onNewsResourcesCheckedChanged = viewModel::updateNewsResourceSaved,
        onNewsResourceViewed = { viewModel.setNewsResourceViewed(it, true) },
        modifier = modifier,
    )
}

collectAsStateWithLifecycle() 可以读成：把 StateFlow 按生命周期安全地收集成 Compose State。当值变化时，读取这个值的 Composable 会重新组合。

5. 内层 ForYouScreen 是更纯的 UI 函数

内层函数不直接知道 ViewModel，只接收状态和事件：

internal fun ForYouScreen(
    isSyncing: Boolean,
    onboardingUiState: OnboardingUiState,
    feedState: NewsFeedUiState,
    onTopicCheckedChanged: (String, Boolean) -> Unit,
    onNewsResourcesCheckedChanged: (String, Boolean) -> Unit,
    onNewsResourceViewed: (String) -> Unit,
)

这种拆法有两个好处：

  • 外层负责状态来源：ViewModel、Hilt、Flow 收集。
  • 内层负责 UI 渲染：给什么状态，就画什么界面；也更容易 Preview 和测试。

6. Loading 不在 newsFeed 里画，而是在屏幕层画

newsFeed 里：

when (feedState) {
    NewsFeedUiState.Loading -> Unit
    is NewsFeedUiState.Success -> {
        items(
            items = feedState.feed,
            key = { it.id },
            contentType = { "newsFeedItem" },
        ) { userNewsResource ->
            NewsResourceCardExpanded(...)
        }
    }
}

Loading -> Unit 的意思是：新闻列表这个 Lazy grid section 在 Loading 时不插入任何新闻 item。

真正的加载动画在屏幕层统一处理：

val isFeedLoading = feedState is NewsFeedUiState.Loading

AnimatedVisibility(
    visible = isSyncing || isFeedLoading || isOnboardingLoading,
) {
    NiaOverlayLoadingWheel(...)
}

这样做的好处是：同步、onboarding、feed 三种加载状态可以共用一个顶部 loading 表达，而不是每个 section 各画各的 loading。

7. 点击新闻卡片后的事件流

在 newsFeed 的 Success 分支里，每个 UserNewsResource 会被渲染成 NewsResourceCardExpanded。点击和收藏仍然走“事件向上”的路线：

onToggleBookmark = {
    onNewsResourcesCheckedChanged(
        userNewsResource.id,
        !userNewsResource.isSaved,
    )
}

传回外层后，最终调用：

viewModel.updateNewsResourceSaved(newsResourceId, isChecked)

然后 ViewModel 调 repository 更新用户数据。数据更新后，userDataRepository.userData 会发出新值，前面那条 Flow 链重新合成新的 UserNewsResource，UI 再重新组合。

这一课的核心

Repository:
Flow<List<UserNewsResource>>

ViewModel:
StateFlow<NewsFeedUiState>

Compose route:
collectAsStateWithLifecycle()

Stateless UI:
when (feedState) {
    Loading -> show no feed items
    Success -> render items
}
