第 7 课：用类型读懂 Flow 操作符

这一课专门解决第 6 课的卡点：Flow.map 和 List.map 不是一回事。读 Flow 链时，先写出每一步的类型，源码会立刻清楚很多。

1. 先拆 feedState

val feedState: StateFlow<NewsFeedUiState> =
    userNewsResourceRepository.observeAllForFollowedTopics()
        .map(NewsFeedUiState::Success)
        .stateIn(
            scope = viewModelScope,
            started = SharingStarted.WhileSubscribed(5_000),
            initialValue = NewsFeedUiState.Loading,
        )

observeAllForFollowedTopics() 的返回类型是：

Flow<List<UserNewsResource>>

这表示这个 Flow 每次发出的值是一个完整的 List<UserNewsResource>。所以这里的 map 不是对列表里的每个 item 调用一次，而是对 Flow 每次发出的“整个列表”调用一次。

2. Flow.map vs List.map

Flow.map

外层变换

Flow<A>.map { a -> b }
// 得到 Flow<B>

每次上游发出一个 A，就把它转换成一个 B。

List.map

内层变换

List<A>.map { a -> b }
// 得到 List<B>

遍历列表里的每个元素，把每个 A 转成 B。

所以这段代码：

Flow<List<UserNewsResource>>
    .map(NewsFeedUiState::Success)

等价于：

Flow<List<UserNewsResource>>
    .map { list ->
        NewsFeedUiState.Success(feed = list)
    }

类型变化是：

Flow<List<UserNewsResource>>
        ↓ map
Flow<NewsFeedUiState.Success>
        ↓ stateIn
StateFlow<NewsFeedUiState>

3. NewsFeedUiState::Success 是什么

NewsFeedUiState.Success 是一个 data class：

data class Success(
    val feed: List<UserNewsResource>,
) : NewsFeedUiState

NewsFeedUiState::Success 是构造函数引用。它可以当成一个函数使用：

(List<UserNewsResource>) -> NewsFeedUiState.Success

因此 .map(NewsFeedUiState::Success) 就是把每次发出的新闻列表包进 Success(feed = list)。

4. 为什么 feedState 不需要 combine

feedState 的上游已经是“当前用户关注主题对应的新闻列表”：

observeAllForFollowedTopics(): Flow<List<UserNewsResource>>

它只需要把列表包装成 UI state，所以用 map 就够了。

onboardingUiState 不同，它需要两个来源：

shouldShowOnboarding: Flow<Boolean>
getFollowableTopics(): Flow<List<FollowableTopic>>

所以它需要 combine。

5. collectAsStateWithLifecycle 的准确读法

你第 6 课第 3 题的方向接近，但要更精确：StateFlow 本身仍然属于 ViewModel；collectAsStateWithLifecycle() 创建的是一个跟随当前 lifecycle 收集这个 Flow 的 Compose State。

当 ForYouScreen 不在活跃生命周期时，收集会停止或暂停；它不会把 ViewModel 里的 StateFlow “关闭”。如果没有 UI 订阅，配合 SharingStarted.WhileSubscribed，上游收集可能在超时后停止。

6. 测试里为什么要 collect

因为 ViewModel 使用：

SharingStarted.WhileSubscribed(5_000)

这表示有订阅者时才收集上游。测试里没有真实 UI，所以需要主动：

backgroundScope.launch(UnconfinedTestDispatcher()) {
    viewModel.onboardingUiState.collect()
}

这相当于模拟 UI 正在订阅这个 state，让上游 Flow 链真正开始工作。
