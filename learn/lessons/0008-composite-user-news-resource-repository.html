第 8 课：组合新闻数据和用户数据

这一课读 CompositeUserNewsResourceRepository。目标是看懂它如何把 NewsResource 和 UserData 合成 UI 需要的 UserNewsResource。

先反馈第 7 课练习

你的类型判断基本正确：

  • userDataRepository.userData 是 Flow<UserData>。
  • .map { it.followedTopics } 后是 Flow<Set<String>>。
  • distinctUntilChanged() 不改变类型，只过滤连续相同的 emission。
  • flatMapLatest 根据 followed topics 切换到不同的内部 Flow，最后仍然输出 Flow<List<UserNewsResource>>。
小校正

distinctUntilChanged() 不是改变“订阅者”，而是改变这条 Flow 往下游发出的内容：如果新的 Set<String> 和上一个连续值相等，就不继续往下发。

1. observeAllForFollowedTopics：先根据用户关注主题决定查询

override fun observeAllForFollowedTopics(): Flow<List<UserNewsResource>> =
    userDataRepository.userData
        .map { it.followedTopics }
        .distinctUntilChanged()
        .flatMapLatest { followedTopics ->
            when {
                followedTopics.isEmpty() -> flowOf(emptyList())
                else -> observeAll(
                    NewsResourceQuery(filterTopicIds = followedTopics)
                )
            }
        }

这段代码的意思是：用户关注的 topic 集合变化时，重新决定新闻列表来源。

空集合

没有关注主题

直接返回 flowOf(emptyList())。没有关注主题，就没有 For You feed。

非空集合

按关注主题查询

用 NewsResourceQuery(filterTopicIds = followedTopics) 查询对应主题的新闻。

2. flatMapLatest：用户关注变了，就换一条新闻流

flatMapLatest 的读法是：上游每发出一个新的 followedTopics，就创建一条新的内部 Flow；如果旧的内部 Flow 还在收集，就取消旧的，只保留最新的。

Flow<Set<String>>
    .flatMapLatest { followedTopics ->
        Flow<List<UserNewsResource>>
    }
// 最终得到 Flow<List<UserNewsResource>>

这很适合“查询条件变化后重新查询”的场景。用户关注主题变化，就是查询条件变化。

3. observeAll：把 NewsResource 和 UserData combine 起来

override fun observeAll(
    query: NewsResourceQuery,
): Flow<List<UserNewsResource>> =
    newsRepository.getNewsResources(query)
        .combine(userDataRepository.userData) { newsResources, userData ->
            newsResources.mapToUserNewsResources(userData)
        }

newsRepository.getNewsResources(query) 提供内容数据，userDataRepository.userData 提供用户状态。两者任意一个变化，都会重新计算 UserNewsResource 列表。

NewsResource

内容数据

标题、链接、发布时间、所属 topics、图片等。

UserData

用户状态

关注了哪些 topics、收藏了哪些新闻、看过哪些新闻。

UserNewsResource

UI 可用模型

内容数据 + 当前用户状态，例如 isSaved、hasBeenViewed、followableTopics。

4. mapToUserNewsResources：这里才是 List.map

fun List<NewsResource>.mapToUserNewsResources(
    userData: UserData,
): List<UserNewsResource> =
    map { UserNewsResource(it, userData) }

注意这里的 map 是 List.map，不是 Flow.map。它会遍历列表里的每个 NewsResource，生成对应的 UserNewsResource。

UserNewsResource(newsResource, userData)

构造时会计算：

  • followableTopics：每个 topic 是否在 userData.followedTopics 中。
  • isSaved：新闻 id 是否在 userData.bookmarkedNewsResources 中。
  • hasBeenViewed：新闻 id 是否在 userData.viewedNewsResources 中。

这节课的核心

CompositeUserNewsResourceRepository 的职责不是从网络或数据库直接取原始数据，而是把多个 repository 的数据组合成更适合 UI/use case 使用的模型。

  1. observeAllForFollowedTopics 根据用户关注主题决定查询哪些新闻。
  2. observeAll 把新闻内容和用户状态合成 UserNewsResource。
  3. UserNewsResource 是 UI 层更容易渲染的模型。
